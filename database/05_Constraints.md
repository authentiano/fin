# 1. Introduction

Constraints ensure that only valid data can be stored in the database. They are enforced by PostgreSQL regardless of the application that accesses the database.

The objectives are to:

- Preserve data integrity.
- Prevent duplicate records.
- Enforce relationships.
- Validate business rules.
- Improve database reliability.

---

# 2. Constraint Philosophy

The database follows these principles:

- Every table has a primary key.
- Every relationship uses foreign keys.
- Business identifiers are unique.
- Required fields cannot be NULL.
- Financial values cannot be negative.
- Invalid status values are prohibited.
- Historical data is protected from accidental deletion.

---

# 3. Constraint Types

The BookRecord System uses:

- Primary Key (PK)
- Foreign Key (FK)
- Unique (UQ)
- Check (CK)
- NOT NULL
- Default (DF)

# 10. Business Constraints

## Customer

CustomerNumber must be unique.

---

Customer email is optional.

If supplied, it must be unique.

---

A customer cannot be deleted while related orders exist.

---

## Orders

OrderNumber must be unique.

---

Order amount cannot be negative.

---

Completed orders cannot be modified.

---

Cancelled orders cannot receive payments.

---

## Payments

Payment amount must be greater than zero.

---

Allocated payment amount cannot exceed payment amount.

---

A refunded payment cannot be allocated.

---

## Deposits

Deposit balance cannot become negative.

---

Deposits cannot exceed recorded amount.

---

## Users

Username must be unique.

---

PasswordHash cannot be empty.

---

Disabled users cannot log in.

---

## Roles

Role names are unique.

---

Administrator role cannot be deleted.

---

## Permissions

Permission names are unique.

---

## Audit Logs

Audit records are immutable.

# 12. Entity Framework Core Configuration

The following constraints shall be configured using the Fluent API.

Examples:

Primary Key

```csharp
builder.HasKey(x => x.CustomerId);
```

---

Unique

```csharp
builder.HasIndex(x => x.Email)
       .IsUnique();
```

---

Required

```csharp
builder.Property(x => x.FirstName)
       .IsRequired();
```

---

Maximum Length

```csharp
builder.Property(x => x.FirstName)
       .HasMaxLength(100);
```

---

Foreign Key

```csharp
builder.HasOne(x => x.Customer)
       .WithMany(c => c.Orders)
       .HasForeignKey(x => x.CustomerId);
```

# 14. Summary

The BookRecord System uses database constraints to enforce business rules, maintain referential integrity, and protect data quality. By combining PostgreSQL constraints with Entity Framework Core Fluent API configuration, the system ensures consistency across the database and application layers while supporting future scalability and maintainability.
